# CareAttend Deployment Guide: Render + Firebase

Date: 2026-06-30

## Recommended Hosting Split

| Component | Host | Reason |
|---|---|---|
| Flask API + website | Render Web Service | Native Flask/Gunicorn deployment, managed HTTPS, health checks, GitHub auto-deploy |
| PostgreSQL | Render Postgres | Same platform/network as backend |
| Flutter web app | Firebase Hosting | Static hosting for `flutter build web`, HTTPS, simple deploy command |

This is for academic prototype/demo hosting only. Do not upload real NHS or
patient-identifiable data.

## 1. Deploy Backend + Website on Render

1. Push the repo to GitHub.
2. In Render, create a new Blueprint from the repo.
3. Render reads `render.yaml` and creates:
   - `careattend-api` web service
   - `careattend-db` PostgreSQL database
4. Before first deploy, set these environment variables in Render:

```bash
CAREATTEND_ADMIN_USER=admin1
CAREATTEND_ADMIN_EMAIL=admin@nhs.test
CAREATTEND_ADMIN_PASSWORD=<strong password>
```

The Blueprint also sets:

```bash
SECRET_KEY=<generated by Render>
DATABASE_URL=<from careattend-db>
FLASK_DEBUG=0
RESET_DEV_CODE=0
CORS_ORIGINS=*
```

After Firebase is deployed, replace `CORS_ORIGINS=*` with the exact Firebase
origin, for example:

```bash
CORS_ORIGINS=https://careattend.web.app,https://careattend.firebaseapp.com
```

Health check:

```bash
curl https://YOUR_RENDER_SERVICE.onrender.com/health
```

Expected response:

```json
{"auth_tables":"ok","database":"ok","model_loaded":true,"status":"ok"}
```

## 2. Deploy Flutter Web App on Firebase

Install Firebase CLI if needed:

```bash
npm install -g firebase-tools
firebase login
```

Build the Flutter web app with the Render backend URL:

```bash
cd /Users/hritiksah/Downloads/CareAttend/care_attend_app
flutter build web --release --dart-define=API_BASE=https://YOUR_RENDER_SERVICE.onrender.com
```

From the repo root, deploy the already-built `care_attend_app/build/web` folder:

```bash
cd /Users/hritiksah/Downloads/CareAttend
firebase init hosting
firebase deploy
```

When Firebase asks:

| Prompt | Answer |
|---|---|
| Public directory | `care_attend_app/build/web` |
| Single-page app rewrite | `Yes` |
| Overwrite `index.html` | `No` |
| GitHub auto deploys | Optional; choose `No` for the simplest manual deploy |

The committed `firebase.json` already points Hosting to
`care_attend_app/build/web` and rewrites all routes to `index.html`.

## 3. Final Production-Demo Checklist

| Check | Command |
|---|---|
| Backend health | `curl https://YOUR_RENDER_SERVICE.onrender.com/health` |
| Flutter app opens | Visit Firebase Hosting URL |
| Login works | Use the Render-created admin account |
| API URL is correct | Browser DevTools network requests go to Render URL, not `127.0.0.1` |
| CORS locked down | `CORS_ORIGINS` set to Firebase origins |
| No real patient data | Use only synthetic/demo data |

## 4. Rollback

Render keeps prior successful deploys in the dashboard. Firebase Hosting keeps
release history and can roll back from the Hosting dashboard.
